## 2. Железо и схема подключения

### 2.1 Компоненты

| Компонент | Кол-во | Назначение |
|-----------|--------|------------|
| ESP32-CAM | 1 | Контроллер, Wi-Fi, веб-сервер |
| PC817 8-канальный модуль | 1 | Оптоизоляция, управление кнопками |
| DHT22 | 1 | Температура и влажность в помещении |
| Резистор 10кОм | 1 | Pull-up для DHT22 |
| Блок питания 5V/2A | 1 | Питание ESP32-CAM |
| Провода Dupont | — | Соединения |

### 2.2 Назначение GPIO ESP32-CAM

```
GPIO  2  → PC817 CH1 → POWER
GPIO  4  → PC817 CH2 → MODE
GPIO 12  → PC817 CH3 → SPEED
GPIO 13  → PC817 CH4 → TIMER
GPIO 14  → PC817 CH5 → SLEEP
GPIO 15  → PC817 CH6 → TEMP+
GPIO 16  → PC817 CH7 → TEMP-
GPIO 33  ← DHT22 DATA (с pull-up 10кОм на 3.3V)  ← было GPIO3 (RX0 конфликт)

Питание:
5V   → ESP32-CAM VCC
5V   → PC817 модуль VCC
GND  → общий
```

### 2.3 Схема PC817 → кнопка кондея

```
ESP32 GPIO ──[1кОм]── IN+ PC817
                       IN- PC817 ── GND

                OUT (коллектор) PC817 ──── Кнопка контакт A
                                           Кнопка контакт B ── GND кондея
```

> Параллельно физической кнопке. При HIGH на GPIO → оптопара замыкается → нажатие кнопки.

### 2.4 Логика нажатия кнопки

- Длительность импульса: **150 мс** (оптимально для мембранных кнопок)
- После нажатия: пауза **300 мс** (защита от bounce)
- Повторное нажатие: не ранее чем через **500 мс**

### 3.5 Пример кода прошивки (main.cpp)

```cpp
#include <Arduino.h>
#include <WiFi.h>
#include <ESPAsyncWebServer.h>
#include <AsyncMqttClient.h>
#include <DHT.h>
#include <ArduinoOTA.h>
#include <ESPmDNS.h>
#include <ArduinoJson.h>

// GPIO пины кнопок
#define PIN_POWER    2
#define PIN_MODE     4
#define PIN_SPEED   12
#define PIN_TIMER   13
#define PIN_SLEEP   14
#define PIN_TEMP_UP 15
#define PIN_TEMP_DN 16
#define PIN_DHT     3

// Параметры нажатия
#define PRESS_DURATION 150  // мс
#define PRESS_COOLDOWN 500  // мс

DHT dht(PIN_DHT, DHT22);
AsyncWebServer server(80);
AsyncMqttClient mqttClient;

struct ACState state = {false, "auto", "hi", 22, false, false, 0, 0, false, false};

void pressButton(int pin) {
  digitalWrite(pin, HIGH);
  delay(PRESS_DURATION);
  digitalWrite(pin, LOW);
  delay(PRESS_COOLDOWN);
}

void publishState() {
  StaticJsonDocument<512> doc;
  doc["power"] = state.power;
  doc["mode"] = state.mode;
  doc["speed"] = state.speed;
  doc["targetTemp"] = state.targetTemp;
  doc["roomTemp"] = state.roomTemp;
  doc["roomHumidity"] = state.roomHumidity;
  String json;
  serializeJson(doc, json);
  mqttClient.publish("home/ac/mystery/state", 0, true, json.c_str());
}

void setup() {
  // Инициализация GPIO
  int pins[] = {PIN_POWER, PIN_MODE, PIN_SPEED, PIN_TIMER, PIN_SLEEP, PIN_TEMP_UP, PIN_TEMP_DN};
  for (int pin : pins) {
    pinMode(pin, OUTPUT);
    digitalWrite(pin, LOW);
  }
  
  dht.begin();
  WiFi.begin(WIFI_SSID, WIFI_PASS);
  MDNS.begin("ac-control");
  ArduinoOTA.begin();
  
  // HTTP API
  server.on("/api/button/power", HTTP_POST, [](AsyncWebServerRequest *req) {
    pressButton(PIN_POWER);
    state.power = !state.power;
    publishState();
    req->send(200, "application/json", "{\"ok\":true}");
  });
  // ... остальные кнопки аналогично
  
  server.serveStatic("/", SPIFFS, "/").setDefaultFile("index.html");
  server.begin();
}

void loop() {
  ArduinoOTA.handle();
  
  // Обновление DHT каждые 30 сек
  static unsigned long lastDHT = 0;
  if (millis() - lastDHT > 30000) {
    state.roomTemp = dht.readTemperature();
    state.roomHumidity = dht.readHumidity();
    publishState();
    lastDHT = millis();
  }
}
```

### Чтение индикаторов — подключение к светодиодам

Подпаиваемся к анодам LED-индикаторов панели. Напряжение на LED ~2-3V — нужен делитель напряжения чтобы не сжечь GPIO ESP32 (макс 3.3V).

**Схема делителя для каждого индикатора:**
```
LED анод ──[10кОм]──┬── ESP32 GPIO (INPUT)
                    │
                 [20кОм]
                    │
                   GND
```
Вых. напряжение: Vled × 20/(10+20) = ~0.67 × Vled ≈ 1.4-2V ✅

**GPIO для индикаторов:**

| Индикатор | GPIO | Описание |
|-----------|------|----------|
| AUTO      | 32   | Режим авто |
| COOL      | 33   | Режим охлаждения |
| FAN       | 34   | Режим вентилятор |
| HI        | 35   | Высокая скорость |
| LOW       | 36   | Низкая скорость |
| FULL WATER| 39   | Бак полный |
| COMP      | 26   | Компрессор работает |

> GPIO 34-39 на ESP32-CAM — только INPUT (нет pull-up), идеально для чтения.

**Логика чтения:**
```cpp
bool readIndicator(int pin) {
  return digitalRead(pin) == HIGH;
}

// В loop() каждые 2 сек:
state.mode = readIndicator(32) ? "auto" : 
             readIndicator(33) ? "cool" : "fan";
state.speed = readIndicator(34) ? "hi" : "low";
state.fullWater = readIndicator(39);
state.compRunning = readIndicator(26);
```

## 11. Архитектурный ревью и правки (24.03.2026)

### 🔴 Правка 1: GPIO конфликты ESP32-CAM

GPIO 2 и GPIO 4 **заняты аппаратно** на ESP32-CAM:
- GPIO 2 — встроенный LED (конфликт с POWER кнопкой)
- GPIO 4 — вспышка камеры (конфликт с MODE кнопкой)

**Обновлённая карта GPIO — кнопки:**

| GPIO | Кнопка | Примечание |
|------|--------|------------|

| 14 | SLEEP | ✅ свободен |
| 15 | SPEED | ✅ свободен |
| 16 | TIMER | ✅ свободен |
| 17 | TEMP+ | ✅ свободен |
| 21 | TEMP- | ✅ свободен |

### 🔴 Правка 2: Pull-down резисторы на индикаторных входах

GPIO 34-39 — только INPUT, без встроенного pull-up/pull-down. Без внешнего резистора вход "плавает" → ложные срабатывания.

**Для каждого индикаторного GPIO добавить:**
```
ESP32 GPIO ──────────────────── к делителю (от LED)
               │
            [10кОм]
               │
              GND
```
Pull-down 10кОм на GND для GPIO 26, 32, 33, 34, 35, 36, 39.

**Итого дополнительных резисторов:** 7 × 10кОм pull-down + 7 × делитель (10кОм+20кОм) = **21 резистор**

### 🔴 Правка 3: LittleFS вместо SPIFFS

SPIFFS устарел и deprecated начиная с ESP-IDF 5.x. Использовать **LittleFS**.

```cpp
// Было:
#include <SPIFFS.h>
SPIFFS.begin(true);
server.serveStatic("/", SPIFFS, "/");

// Стало:
#include <LittleFS.h>
LittleFS.begin(true);
server.serveStatic("/", LittleFS, "/");
```

В `platformio.ini`:
```ini
board_build.filesystem = littlefs
```

### 🔴 Правка 4: Нажатие кнопки через FreeRTOS Queue

`delay()` в колбэке AsyncWebServer блокирует весь веб-сервер. Нужна очередь задач.

```cpp
// Очередь для нажатий
QueueHandle_t buttonQueue;

struct ButtonPress {
  int pin;
  int duration;
};

// При старте:
buttonQueue = xQueueCreate(10, sizeof(ButtonPress));

// Отдельная задача для нажатий:
void buttonTask(void* param) {
  ButtonPress press;
  while (true) {
    if (xQueueReceive(buttonQueue, &press, portMAX_DELAY)) {
      digitalWrite(press.pin, HIGH);
      vTaskDelay(press.duration / portTICK_PERIOD_MS);
      digitalWrite(press.pin, LOW);
      vTaskDelay(300 / portTICK_PERIOD_MS);  // cooldown
    }
  }
}

// В setup():
xTaskCreate(buttonTask, "ButtonTask", 2048, NULL, 1, NULL);

// В HTTP handler (не блокирует сервер):
server.on("/api/button/power", HTTP_POST, [](AsyncWebServerRequest* req) {
  ButtonPress press = {PIN_POWER, 150};
  xQueueSend(buttonQueue, &press, 0);
  req->send(200, "application/json", "{\"ok\":true}");
});
```
